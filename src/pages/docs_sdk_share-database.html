Docs : SDK : shareDatabase

shareDatabase lets you share a database with another user.

    
      userbase.shareDatabase({
        databaseName: 'example-database-name',
        username: 'alice'
      }).then(() => {
        // Alice can now access the database
      }).catch((e) => console.error(e))

Parameters

  • databaseName [string] - The database name to share.
  • databaseId [string | optional] - The database to share's unique identifier. Can be used in place of databaseName.
  • username [string] - The username to share the database with.
  • requireVerified [boolean | optional] - Requires the user be verified via verifyUser. Defaults to true. This ensures the user is sharing the database directly with the expected user, and not a man-in-the-middle.
  • readOnly [boolean | optional] - Sets the recipient's access permissions to read only. Defaults to true.
  • resharingAllowed [boolean | optional] - Sets the recipient's permissions to reshare the database. Defaults to false.

Notes

  • You can provide either a databaseName or a databaseId, but not both.
  • To access a database that has been shared with a user, a user can find the database's databaseId via getDatabases and then provide that databaseId to openDatabase.
  • A user can verify another user via verifyUser at any point, either before or after sharing a database. Verifying a user after sharing a database will enable the verifier to make sure they shared the database directly with the expected user, and not a man-in-the-middle.
  • If a user deletes their account via deleteUser and they are the owner of a database, that database will be deleted even if it has been shared with other users.
